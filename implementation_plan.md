# SensAI Next-Gen Adaptive Learning Implementation Plan

This document outlines a modular, graceful approach to adding the **Adaptive Learning Engine**, **Discussion Chatbot**, and **Universal Score Engine** into the existing SensAI architecture. 

Currently, SensAI uses FastAPI + Pydantic (Backend) and Next.js + React (Frontend). The learning structure hierarchy is `Course > Milestone > Task (Blocks)`.

## User Review Required

> [!IMPORTANT]
> Since we are building an intelligent tracking and adaptive routing system, we need to decide on **Data Retention**:
> - How long should detailed tracking signals (scroll depth, pause counts) be kept before aggregation? Raw data scales quickly.
> - Do you want the dynamic generation of course paths to store virtual tasks or permanently modify the user's cohort course structure? (Recommendation: Virtual/Transient tasks visible only to the specific user).

> [!WARNING]  
> Modifying the frontend to capture scrolling and tab-focus (Attention Signals) could impact frontend performance if not debounced correctly. We will plan to use lightweight React hooks and `navigator.sendBeacon`.

---

## Proposed Changes

---

### Phase 1: Universal Score Engine & Signals Tracking
We start here because the Chatbot and Adaptive Engine require the data and derived states generated by the Score Engine.

#### Backend
- **[NEW] `src/api/signals.py` (Routes)**
  - Endpoints to ingest raw telemetry data from the frontend (e.g., `POST /signals/log`).
- **[MODIFY] `src/api/models.py`**
  - Add `UserLearningSignal` model: tracks `task_id`, `user_id`, `consumption_metrics` (scroll depth, completion %), `engagement_metrics` (pauses, seek backs), `attention_metrics` (tab focus, idle time).
  - Add `UserLearningProfile` model: stores the derived state (`learning_category`: visual/theory, `status`: Learned/Partially Learned/Struggling/Passive).
- **[NEW] `src/api/cron.py` / Task Queue**
  - Add a scheduled or background worker (`compute_learning_status`) to aggregate raw `UserLearningSignal` entries, apply the probabilistic categorization, and check the "humanizer" (anti-faking check). Update `UserLearningProfile`.

#### Frontend
- **[NEW] `src/hooks/useContentTelemetry.ts`**
  - React hook attached to Article / Video blocks. Attaches listeners for `scroll`, `visibilitychange` (tab focus), `pause`, `seeked`, `play`.
  - Uses throttling (e.g., every 5 seconds) to send data to the backend.
- **[MODIFY] `src/components/Blocks/VideoBlock.tsx` & `ArticleBlock.tsx`** (Or equivalent Block renderer)
  - Integrate the telemetry hook to start logging signals seamlessly.

---

### Phase 2: Discussion Chatbot / AI Tutor
The chatbot acts as the interactive checkpoint that also feeds data into the Score Engine based on the learner's conversational performance.

#### Backend
- **[MODIFY] `src/api/chat.py` & `src/api/ai.py`**
  - Expand the existing AI logic (which already has `AIChatRequest` and `ChatResponseType`) to include conversational flows: `SUMMARIZE_VIDEO`, `TEACHER_QA`, `COMPETITIVE_QUIZ`, `RECALL_SPACED_REPETITION`, `3_2_1_STRATEGY`.
  - Add an evaluation prompt behind the scenes: when the user summarizes or answers, the AI rates the response and feeds the score into the `UserLearningSignal` table.
- **[MODIFY] `src/api/models.py`**
  - Add fields to `ChatMessage` or create a `ChatAssessment` to link chat session quality to the `Task`.

#### Frontend
- **[NEW] `src/components/Chatbot/PostTaskChatbot.tsx`**
  - A modal or side-panel component that triggers upon completing a video/article block.
  - Initial prompt configuration based on the strategy chosen (e.g., "Great job finishing! Can you summarize 3 things you learned?").
- **[MODIFY] `src/app/courses/[courseId]/tasks/[taskId]/page.tsx`** (or equivalent task view)
  - Inject the `PostTaskChatbot` component. Trigger its opening when the user marks the material as complete or hits 100% telemetry completion.

---

### Phase 3: Adaptive Learning Engine — Evolving Paths
This phase connects the computed `UserLearningProfile` and `ChatAssessment` to the user's specific journey.

#### Backend
- **[MODIFY] `src/api/task.py` & `src/api/course.py`**
  - **Dynamic Task Serving:** Currently, tasks are static per milestone. We will introduce `get_next_adaptive_task(user_id, milestone_id)`.
  - If `UserLearningProfile` = *Struggling*, the system generates or fetches a "Reinforcement Module" `Task` (alternate explanation or easier quiz).
  - If *Skipped/Passive* or *Faking*, the system introduces a mandatory "Delayed Re-test" task before unlocking the next milestone.
- **[NEW] `src/api/adaptive.py`**
  - Logic engine prioritizing rules: 
    - Rule 1: Conflicting signals (e.g., high quiz score but low watch time + low chat score) -> trigger Chatbot Verification.
    - Rule 2: Cold-Start -> Give default cohort path until minimum 3 tasks are completed.

#### Frontend
- **[MODIFY] `src/components/Course/MilestoneList.tsx`** (or equivalent timeline component)
  - Re-fetch the milestone structure dynamically as the user completes tasks, revealing newly injected adaptive tasks with a "Recommended for You" or "Review" tag so they gracefully understand why their path changed.

## Open Questions

1. **AI Chatbot Modality:** Do you want the Discussion Chatbot to support Audio inputs (voice summary) or strictly text for now?
2. **Adaptive Content Generation:** When a user is struggling, do you want SensAI to use the LLM to *generate entirely new alternative content blocks on the fly*, or route them to *pre-written alternative modules* created by the human instructor?
3. **Faking Heuristics:** Should users caught by the "humanize check" (e.g., immediately scrubbed to the end of a 20m video) be blocked from proceeding, or just silently flagged for the instructor?

## Verification Plan

### Automated Tests
- Pytest for Signal Aggregation logic to ensure it accurately labels (Learned vs Passive) based on mock telemetry data.
- Unit tests for Chatbot prompt generation (ensuring 3-2-1 strategy prompts correctly).

### Manual Verification
- Simulate a "Passive" learning style on the frontend (fast scrolling, skipped video). Verify backend flags user as Passive.
- Complete a video, trigger Chatbot, provide a weak summary, and verify the Adaptive Engine injects a reinforcement task into the timeline.
